# Kingdom Manifesto

## What We Saw

The first Kingdom MVP lived inside Minecraft.

That world mattered because it gave us proof, not fantasy. We saw that a small system of agents could coordinate, remember, reflect, and improve when given:

- clear roles
- shared state
- live observability
- reusable skills
- human direction

Minecraft was the first environment where that hope became visible.

## What Changes Now

Kingdom is no longer defined by the world where the first proof happened.

Kingdom now exists to build systems in the real world:

- systems that preserve context
- systems that help humans think and decide
- systems that let agents collaborate instead of improvising blindly
- systems that compound learning over time
- systems that turn hope into repeatable work

We are not trying to keep a sandbox alive.
We are trying to create an operating system for meaningful human-agent collaboration.

## The Mission

Kingdom exists to help humans and agents build useful things together with:

- disciplined planning
- grounded knowledge
- transparent execution
- rigorous verification
- compounding memory

## The Operating Model

### Planning Plane

The planning plane is where intention becomes structure.

BMAD provides:

- briefs
- PRDs
- architecture
- stories
- sprint planning
- review checkpoints

### Knowledge Plane

The knowledge plane is where context becomes durable.

- Obsidian stores living notes, sessions, decisions, and internal memory
- NotebookLM stores grounded sources and reference knowledge
- GoT converts isolated notes into connected understanding

### Execution Plane

The execution plane is where work becomes software and systems.

- Claude Code
- Codex
- Antigravity
- local agents
- shared Blackboard channels

### Governance Plane

The governance plane is where power stays accountable.

- tests
- reviews
- verification loops
- safety rules
- observability
- human approval at the right boundary

## The Principles

1. Context before action
2. Decomposition before delegation
3. Verification before completion
4. Memory before repetition
5. Taste before scale
6. Real-world usefulness over internal cleverness
7. Compounding systems over one-off output

## The Role of Minecraft

Minecraft remains part of Kingdom as:

- origin story
- proof of concept
- optional adapter for sandbox experiments

It is not the product boundary anymore.

## The Standard

Every new feature, workflow, or agent should answer:

- Does it improve shared context?
- Does it reduce repeated explanation?
- Does it help multiple agents coordinate safely?
- Does it create reusable knowledge?
- Does it increase the quality of real-world output?

If not, it does not strengthen Kingdom.

## The Responsibility

Kingdom is built with unusual leverage.

That means unusual responsibility:

- to keep the system honest
- to keep the knowledge grounded
- to keep the execution reviewable
- to keep the human in command of direction and judgment

The objective is not autonomy for its own sake.
The objective is to build a system that helps turn the world toward hope, one disciplined layer at a time.
