되새김질 Engine — 4 Stomachs

01
Rumen (반추위)
Raw Experience Intake
Builder의 모든 행동(성공/실패)이 buffer에 들어온다. feed()와 feedFailure()로 구분. 실패는 nutrition multiplier 1.5x — 뼈는 영양가가 더 높다.
02
Reticulum (벌집위)
Pattern Filtering
error type별로 경험을 분류. 같은 에러에 어떤 skill들이 사용됐는지, 성공/실패 빈도 추적. co-occurrence (함께 쓰인 스킬) 감지.
03
Omasum (겹주름위)
Insight Extraction
3가지 인사이트 추출: effective_skill (잘 되는 스킬), co_occurrence (시너지 조합), failure_pattern (반복 실패 → 새 스킬 필요).
04
Abomasum (주름위)
Final Digestion → Action
인사이트를 행동으로: 링크 강화, XP 추가, 실패 패턴은 rumination note로 기록. 충분히 강한 링크는 compound skill 생성 트리거.

Gomguk Mode (곰국 모드)

뼈까지 푸욱 고아서 영양분을 뽑아낸다

deepRuminate(): 30분마다 전체 Zettelkasten을 재분석.
- 링크된 but 아직 compound 아닌 skill 쌍 → compound 후보 발견
- 5번 이상 사용, 성공률 50% 미만, 소화 3회 미만 → 재소화
- dormant skill에 rumination note 추가 ("다시 봐야 할 기술")

XP Rules

  • Success → +3 XP (effective combat)
  • Failure → +1 XP (even bones have value — 뼈도 곰국)
  • Compound inherit → 50% parent XP (inherited knowledge)
  • Deprecation → XP freezes, moved to deprecated/

Compound Skill Creation

Two skills that co-occur 5+ times with 70%+ success automatically merge into a compound skill. Like forging two weapons into a legendary blade.

Graph of Thought — 4 Reasoning Strategies

🔍

1. Synergy Discovery

Find unlinked skills that SHOULD be combined. Scores based on: shared error types, shared tags, complementary tiers, both having high success rates.

⚔️

2. Optimal Builds

RPG character build optimization. Start from high-XP skills, greedily expand through strongest links. Returns top 5 skill combos ranked by buildStrength.

🕳️

3. Gap Analysis

Find error types with no skills or only Novice-level coverage. Critical gaps (zero skills) vs moderate gaps (weak skills). Triggers new skill creation.

📈

4. Evolution Paths

For each skill, predict: uses needed to reach Master, estimated game sessions, compound potential with neighbors, current bottleneck (low success vs needs more usage).

Auto-Memory Wiring

zettelkasten-hooks.js intercepts existing agent methods — zero changes needed to builder/leader/pipeline code.

  • builder._tryLearnedSkill() → records usage + co-occurrence in Zettelkasten, feeds Rumination
  • builder._selfImprove() → feeds failures as gomguk (1.5x nutrition)
  • leader.triggerGroupReflexion() → triggers full GoT reasoning cycle
  • pipeline.deploySkill() → creates atomic Zettelkasten note
  • pipeline.updateSuccessRate() → mirrors XP to Zettelkasten, auto-deprecate

Pub/Sub Events

  • zettelkasten:tier-up → celebration broadcast to all agents
  • zettelkasten:compound-created → logged + vault synced
  • rumination:digested → after 5 digestions, triggers GoT reasoning
  • got:reasoning-complete → summary stats published
