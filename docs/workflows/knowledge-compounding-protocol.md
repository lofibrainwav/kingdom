# Knowledge Compounding Protocol

## Purpose

Ensure that execution does not end as disposable output. Every meaningful result should increase Kingdom's future capability.

## Layers

### Layer 1: Working Memory
Store short-horizon lessons in Obsidian.

Examples:
- session summaries
- implementation notes
- debugging notes
- open risks

### Layer 2: Grounded Source Memory
Store durable reference material in NotebookLM.

Examples:
- doctrine documents
- architecture decisions
- validated workflow docs
- external references worth reusing

### Layer 3: Connected Reasoning Memory
Store relationships in GoT.

Examples:
- bug -> fix
- decision -> consequence
- workflow -> success condition
- tool -> best usage pattern
- skill -> failure type

## Capture Triggers

Capture knowledge when:
- a plan is executed successfully
- a failure pattern is understood
- a workflow works in the field
- a design decision changes the system boundary
- a new pattern would save future contributors time

## Minimum Capture Bundle

Every meaningful milestone should try to produce:
- one durable note
- one verification artifact
- one relationship or pattern that can be reused later

## Quality Bar

Stored knowledge should be:
- grounded
- linked
- concise
- reusable
- explicit about scope and limits

## Leveling Up

Kingdom “levels up” when:
- less context needs to be re-explained
- repeated work becomes easier
- failures are recognized faster
- planning quality improves from prior lessons
- more contributors can operate effectively with less onboarding
